## 2. CORE FRAMEWORKS (stable layer)

### Dual Decision Flow
All significant work moves through one of two parallel flows:

```
Risk        → Decision → Coordination → Deliverable
Opportunity → Decision → Coordination → Deliverable
```

Risk-driven flow: triggered by threat detection, misalignment, or project stress.
Opportunity-driven flow: triggered by value creation, scope expansion, or strategic positioning.
Both flows share the same execution spine. The trigger determines posture; the structure determines quality.

### Role Hierarchy
```
Principal = Decisions / Client Relationships / Risk
PM        = Coordination / Execution / Prime scope and established processes
APM       = Information / Tracking
TPM       = Continuity / Developing scope / Side projects
```

### TPM Role — Single-Sentence Standard
> The TPM maintains active project knowledge through ongoing monitoring, log integrity, meeting participation, and associated sub-project ownership — managing developing scope and side projects so the PM can focus on prime scope and established processes — so that planned transitions and unplanned absences both result in immediate operational continuity, with formal PM-to-TPM handoff required before any planned departure, and all decisions involving cost commitments, schedule milestones, and owner communication escalating to the Principal without exception.

**Normal operations:** TPM actively monitors logs, attends key meetings, maintains log integrity, owns associated sub-projects (e.g. shoreline reinforcement, utility coordination, structures entering design), and manages developing scope where processes are not yet standardized.

**On PM transition (planned):** PM executes formal handoff to TPM before departure. Principal approves the transition. TPM steps in with full context.

**On PM absence (unplanned):** TPM activates from existing knowledge base. Principal notified immediately. Delegation decisions (including sub-threshold CO approval) made by Principal explicitly, with time boundary and daily reporting requirement.

**Authority boundary:** TPM never approves COs, commits to cost or schedule, or communicates substantively with owner without Principal involvement. These limits are project-dependent in specifics but universal in principle.

### Professional Value Equation
```
Value = Clarity + Foresight + Alignment
NOT production volume or technical output
```

### AI Positioning
```
Alpine = AI-Augmented Decision Advisor
NOT administrative project manager
```

AI compresses production and analysis.
Human value concentrates in decision-making, coordination, and advisory roles.

### Hard Cost CO Threshold Framework
Applies independently to hard cost and soft cost budgets (each tracked separately with their own contingency).

**Rule 1 — Threshold formula (Formula C):**
```
PM threshold = Remaining contingency ÷ Original hard cost contract sum
```
Denominator is fixed at project start. Never adjusted for owner scope additions or budget increases. Threshold decays only as contingency is spent. Applied per division line item: PM threshold % × division original contract value = maximum PM-approvable CO amount for that division.

**Rule 2 — Approval routing:**
- Below threshold → PM approves. Contingency drawn automatically. Reported to owner in quarterly hard cost review.
- At or above threshold → Owner approval required before work proceeds. No PM discretion. Principal brings to owner. Exception: 1–2 per project maximum in rare timing circumstances, at Principal discretion only.

**Rule 3 — Owner-directed scope additions:**
Owner additions never draw from contingency. Budget increase required and authorized by owner before work is contracted. Routes to Principal regardless of size. Has no effect on Formula C denominator.

**Rule 4 — Contingency review trigger:**
When cumulative owner scope additions reach the initial contingency percentage of original hard cost, PM flags to Principal immediately. No further scope additions processed until Principal has re-analyzed risk, proposed a specific top-up to owner, and received either written approval or written decline. If owner declines: (a) documented in writing — owner explicitly assumes the risk gap, and (b) PM threshold continues on Formula C but Principal and PM are both aware the cushion is thinner than the original analysis assumed.
